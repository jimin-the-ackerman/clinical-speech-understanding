WER and cpWER, hands on

A short tutorial. You will compute both metrics yourself: type transcripts and watch the score, step the alignment algorithm, and reassign speakers to see attribution cost. Each lesson ends with a quick check.

by the end you can
  1. compute WER from an alignment, and name each edit
  2. explain how the aligner decides which edit is which
  3. compute cpWER and say why it moves above or below WER
  4. pick micro over macro averaging, and know the trade
domain clinical STT prereq none time ~10 min
Lesson 1

Compute a WER

Goal: read a word alignment and turn it into a number.

Word Error Rate lines the hypothesis up against the reference and counts three kinds of edit, over the number of reference words:

WER= (S + D + I) / Nref  # substitutions, deletions, insertions / reference words
word substitution +word insertion word deletion

Try it / live WER

Edit either line. The alignment and score recompute as you type. (Whitespace-tokenized and lowercased here; real WER also normalizes numbers and spelling, a separate step.)

check yourself
Reference the pain is worse today, hypothesis the pain worse today. What is the WER?
0.20. Only is is dropped, a single deletion, over five reference words: 1/5 = 0.20. Paste it into the calculator above to confirm.
Lesson 2

How the aligner decides

Goal: see where S, D, and I actually come from.

The aligner does not "recognize" a substitution. It finds the minimum-cost alignment between the two word sequences by dynamic programming, then reads the edit types off the path. Ref words label the rows, hypothesis words the columns; each cell is the cheapest edit cost to reach it. Step the backtrace from the bottom-right corner. The direction of each move is the edit it records.

↖ match ↖ substitution ↑ deletion ← insertion

When several paths tie for the minimum cost, the split of S vs D+I is not unique (a substitution costs the same as a deletion plus an insertion). The total, and so the WER, is always well defined. This example was chosen to have a single optimal alignment.

Lesson 3

Add the speakers: cpWER

Goal: compute cpWER and feel where its extra cost comes from.

WER is speaker-blind. concatenated permutation WER concatenates each speaker's words into one string per speaker, tries every permutation pairing the model's speaker clusters to the reference speakers, keeps the lowest-WER pairing, then scores those streams as one pool. A word in the wrong mouth costs twice: deleted from the right stream, inserted into the wrong one.

Try it / attribution cost

The doctor said take two tablets, the patient said okay thanks. The model got every word right. Click a word to change which speaker it is attributed to, and watch cpWER.

check yourself
The model transcribes every word correctly but tags two of the doctor's words as the patient's. What happens to the scores?
cpWER rises, WER unchanged. WER never reads speaker labels, so perfect words score 0. cpWER charges the misattribution: the two words are deleted from the doctor stream and inserted into the patient stream.
Lesson 4

The twist: cpWER can go lower

Goal: read the sign of the WER-to-cpWER gap as a diagnostic.

From real PriMock57 audio. When two people talk at once the reference orders overlapping words by start time, but the model linearizes them however it likes. Plain WER punishes every cross-speaker reordering. cpWER splits the speakers apart before comparing, so it never has to match the interleaving, and forgives the reordering.

Example / perfect words and speakers, overlapping turns

plain WER0.50 cpWER0.00
refanyrightpainhere
hypanypainrighthere
reference interleaves by time, model groups by speaker   WER = 2/4 = 0.50
Dr refanypain
Dr hypanypain
Pt refrighthere
Pt hyprighthere
each stream is internally ordered   cpWER = 0/4 = 0.00

So the gap has a sign, and the sign is a diagnostic:

cpWER > WERthe diarizer is confusing speakers; attribution errors stack on recognition errorsbad
cpWER < WERthe diarizer is solid; the extra WER was mostly overlap reordering, which cpWER forgivesgood

Our n=3 Soniox probe landed cpWER 0.12 against flat 0.14, the lower-cpWER case. That quietly suggests attribution is working. It is not a claim that diarization improved transcription.

check yourself
Two people talk over each other and the diarizer separates them well. Compared with flat WER, cpWER is usually...
Lower. Flat WER penalizes the cross-speaker word order the model cannot match on overlaps. cpWER splits speakers first and forgives it, so good attribution plus overlap lands cpWER below WER.
Lesson 5

Averaging across files: micro vs macro

Goal: combine per-file scores without letting a short file lie to you.

The permutation is chosen per file. The only question is how you average across a dataset, and the two ways can diverge sharply.

macro= (1/N) · Σ ( erri / wordsi )  # divide each file, average the rates
micro= Σ erri / Σ wordsi     # pool everything, divide once

Three consultations, one of them tiny and noisy:

consultationerrorsref wordscpWER
day1_consult0121000.020
day1_consult0231500.020
day1_consult03 (tiny)4100.400
macro, mean of rates0.147
micro, pooled 9/26092600.035

Macro (0.147) lets the 10-word file count as much as the 150-word one. Micro (0.035) weights every file by its words, so the tiny file barely registers. Micro is the standard (CHiME-6, MeetEval) and matches how the leaderboard already pools flat WER, so cpWER and WER stay comparable.

check yourself
A dataset is mostly long clean consultations plus one tiny noisy clip. Which average does the clip swing most?
Macro. It weights every file the same, so a 10-word clip counts as much as a 2,000-word consultation. Micro weights by words, so the clip barely moves it. That is why micro is standard.
